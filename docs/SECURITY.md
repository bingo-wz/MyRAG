# 本地运行与文件防护

## 本地信任边界

MyRAG 当前定位为单用户本地工具，不提供登录、用户、角色或领域级权限控制。前端直接调用后端 API，所有功能默认可用；`createdBy`、`reviewer` 和问答操作者按本地输入或固定的“本地用户”记录。

Compose 只把前端、MinIO Console 和 Milvus 端口绑定到 `127.0.0.1`，后端、PostgreSQL、Kafka、etcd 和 MinIO API 不直接暴露到宿主机网络。该边界只适用于个人电脑：不要通过路由器端口转发、反向代理、隧道或公网服务器暴露本项目。

如果以后需要多人或远程访问，必须先重新设计身份认证、权限模型、租户/领域隔离、TLS、密钥托管和审计策略，不能直接复用当前无登录模式。

## 操作审计

生产拓扑仍记录 `/api/**` 下的 `POST`、`PUT`、`PATCH`、`DELETE` 请求，操作者统一记为“本地用户”，并保存方法、路径、响应状态、来源地址、User-Agent 和时间。审计表不记录请求体、文件内容或模型密钥。

本地审计用于排查误操作和功能问题，不具备多用户归责或防抵赖能力。

## 文件防护

当前项目不集成独立杀毒服务，文件导入依靠扩展名白名单、实际 MIME 校验、单文件/单批容量限制、最大提取字符数、正文质量校验和默认 120 秒解析硬超时降低风险。

解析运行在独立 Worker 进程和可取消任务中。容器使用只读根文件系统、非 root 用户、能力裁剪、临时目录配额和资源上限。即使仅在本机使用，也不建议导入来源不明的文件；如未来允许不可信用户上传，应增加独立解析沙箱和组织认可的恶意文件检测能力。

## 本地检查清单

- 管理页面只能通过 `http://127.0.0.1:3000` 或 `http://localhost:3000` 访问。
- 不开放数据库、Kafka、MinIO API、etcd 或后端容器端口。
- `.env` 和模型 API Key 不进入 Git，文件权限限制为当前用户可读。
- 不执行来源不明的宏、脚本或附件；损坏件、加密件和超限件应被拒绝或进入可追踪失败状态。
- 审计日志不得包含请求正文、知识正文、原文件或模型密钥。
